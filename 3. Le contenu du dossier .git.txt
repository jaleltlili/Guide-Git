Le dossier .git héberge tout le ontenu du dépôt utilisé par Git.

Visualiser les dossier et les fichiers présents dans ce dossier:
ls -la .git

Le système d'exploitation nous affiche une liste de fichiers et de dossiers:
$ ls -la .git
total 11
drwxr-xr-x 1 Jalel 197121   0 juin  10 17:04 ./
drwxr-xr-x 1 Jalel 197121   0 juin  10 17:04 ../
-rw-r--r-- 1 Jalel 197121 130 juin  10 17:04 config
-rw-r--r-- 1 Jalel 197121  73 juin  10 17:04 description
-rw-r--r-- 1 Jalel 197121  23 juin  10 17:04 HEAD
drwxr-xr-x 1 Jalel 197121   0 juin  10 17:04 hooks/
drwxr-xr-x 1 Jalel 197121   0 juin  10 17:04 info/
drwxr-xr-x 1 Jalel 197121   0 juin  10 17:04 objects/
drwxr-xr-x 1 Jalel 197121   0 juin  10 17:04 refs/

Chacun de ces fichiers ou dossiers contients des éléments que Git utilise pour suivre notre code.

Voici qqs fichiers et dossiers et leur contenu:
- HEAD : contient la référence du commit à partir duquel on travaille. C'est une référence qui est indépendante de la branche sur laquelle nous sommes situés.
HEAD désignera le commit le plus récent d'une branche et par défaut le commit le plus récent de la branche courante.
- config: ce fichier contient les éléments de configuration et les alias propres au dépôt.
- info: ce dossier contient un fichier exclude utilisé pour demander à Git d'ignorer les fichiers spécifiés. Une autre possibilité existe pour ignorer des fichiers...
- objects: contient tous les objets de notre projet, c'est à dire que c'est dans ce dossier que seront placées toutes les informations concernant nos fichiers, dossiers, commits, ou tout autre objet que Git a besoin de traiter.
- refs: ce dossier comprend les références qui pointent vers des commits. Ces références sont en réalité soit des branches, soit des tags...
